Installing Eagle on 64-bit Ubuntu

Eagle CAD for Linux is a 32-bit application. Here's a quick rundown on the extra loops to jump through when installing Eagle on a 64-bit operating system

If you run the install script as prompted like this: 


chmod +x eagle-lin-6.5.0.run
sudo ./eagle-lin-6.5.0.run <directory-name-here>

You'll get an error when you try to run the executable telling you that there is no such file or directory. The solution? We'll need to install the 32-bit libraries that Eagle needs before we can successfully run it.

For a quick listing on exactly what we need, we can run ldd on the binary. Assuming I tried to install Eagle to the /opt directory, I'd input 


ldd /opt/eagle-6.5.0/bin/eagle

and get: 


    linux-gate.so.1 =>  (0xf7709000)
    libXrender.so.1 => not found
    libXrandr.so.2 => not found
    libXcursor.so.1 => not found
    libfreetype.so.6 => not found
    libfontconfig.so.1 => not found
    libXext.so.6 => not found
    libX11.so.6 => not found
    libdl.so.2 => /lib32/libdl.so.2 (0xf76ef000)
    libXi.so.6 => not found
    libpthread.so.0 => /lib32/libpthread.so.0 (0xf76d3000)
    librt.so.1 => /lib32/librt.so.1 (0xf76c9000)
    libssl.so.1.0.0 => not found
    libcrypto.so.1.0.0 => not found
    libstdc++.so.6 => /usr/lib32/libstdc++.so.6 (0xf75e0000)
    libm.so.6 => /lib32/libm.so.6 (0xf759a000)
    libgcc_s.so.1 => /usr/lib32/libgcc_s.so.1 (0xf757c000)
    libc.so.6 => /lib32/libc.so.6 (0xf73d0000)
    /lib/ld-linux.so.2 (0xf770a000)
    libz.so.1 => not found
Presto, now we just need to install the 32-bit versions of this libararies by specifying the 32-bit version from the packages that these libraries come from. 

sudo apt-get install libxrender1:i386                                           
sudo apt-get install libxrandrr2:i386                                            
sudo apt-get install libxcursor1:i386                                            
sudo apt-get install libfreetype6:i386                                           
sudo apt-get install libfontconfig1:i386                                         
sudo apt-get install libxi6:i386                                                 
sudo apt-get install libssl1.0.0:i386

Now, running Eagle should work as expected. If not, install the remaining missing libraries from their native packages in the format above.
