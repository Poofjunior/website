FPGA Design Techniques 101

Part 0: Duplicating Digital ICs

The Preamble

FPGAs are powerful devices, allowing you to synthesize your own digital logic circuits and upload them to a piece of hardware that lets you use your circuit in the real world. Someone else has said this better than I can: "With micro[controller]s you tell them what to DO, with FPGAâ€™s you tell them what to BE." An FPGA is a chip that can be programmed to "become" another chip, a number of chips, or a custom-made chip altogether. This is the power of programmable logic; you, the designer (or hobbyist?) can synthesize components that do exactly what you want to them do.

There is a catch, unfortunately. SystemVerilog, VHDL, and other programming languages used to program FPGAs are slightly less intuitive that many other languages. C and Python are primarily procedural programming languages. C++ and Java are primarily object-oriented programming languages. SystemVerilog and VHDL are descriptive programming languages. If you haven't yet tried programming in a descriptive language (Prolog anyone?), it's slightly different in mindset. This tutorial is the first of a few tutorials oriented towards adjusting one's mindset for FPGA programming.

Duplicating Chips

To enter the mindset of FPGA programming, let's start by copying an existing digital Integrated Circuit (IC), the STP16C596. This chip is a serial-to parallel converter. The basic idea? Serial data arrives on one pin and gets written to the pins on the chip (one bit in the serial data per pin on the microcontroller).
